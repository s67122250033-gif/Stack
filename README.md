5. สถานะของ Stack หลังจาก Undo: [Type Data, Type Structure, Delete Structure]

LIFO (Last-In, First-Out): หมายความว่า ข้อมูลที่ถูกนำเข้าสู่โครงสร้างเป็น "ลำดับสุดท้าย" จะต้องเป็นข้อมูลตัว "แรก" ที่ถูกนำออกไปเสมอ

การประยุกต์ใช้ในระบบ Undo: จากตัวอย่าง คำสั่ง Type Java ถูกป้อนเข้ามาเป็นตัวสุดท้าย (อยู่บนสุดของ Stack) เมื่อสั่ง Undo (เรียกฟังก์ชัน pop()) ข้อมูลนี้จึงหลุดออกไปก่อน ทำให้ระบบสามารถย้อนกลับไปยังสถานะล่าสุดของผู้ใช้งานได้อย่างถูกต้องการประยุกต์ใช้ในระบบ Undo: จากตัวอย่าง คำสั่ง Type Java ถูกป้อนเข้ามาเป็นตัวสุดท้าย (อยู่บนสุดของ Stack) เมื่อสั่ง Undo (เรียกฟังก์ชัน pop()) ข้อมูลนี้จึงหลุดออกไปก่อน ทำให้ระบบสามารถย้อนกลับไปยังสถานะล่าสุดของผู้ใช้งานได้อย่างถูกต้อง

Time Complexity เบื้องต้น: การทำ push() และ pop() มี Time Complexity เป็น $O(1)$ เสมอ เพราะเป็นการจัดการข้อมูลที่จุดยอด (Top) ของ Stack โดยตรง ไม่ต้องไล่หาข้อมูลตัวอื่น
